### What I learned
- Before any virtual machine can run, secure boot must be turned off and virtualization turned on.
- I learnt of the various navigation commands about 8 of them. `cd ..` moves one step up into the parent directory towards the root, `cd /` moves to the root directory of the entire file filesystem while `cd ~` moves to the home directory from anywhere
-  I also learnt the reason for hidden files isn't due to sensitive information but for personal configuration. They're hidden so you don't modify or accidentally delete them and so `ls` output/file view stays clean. This matters for security because attackers and defenders both know that dot files contain juicy configuration information — SSH keys in `.ssh/`, shell history in `.bash_history`, git credentials in `.gitconfig`.
- / is the root and shows the overview of the system
- /etc contains configuration files (passwords, network settings, service configs) that are a goldmine for attackers
- /home is the user home directory and shows the various users in the system including root.
- /var/log contains logs that show all that's happened in the system.
- /tmp stores temporary file and is used by attackers to drop their tools since it's usually writable for everyone
- /usr/bin is the user's tools or program e.g python3. This is useful for attackers as they check what programs you have that they can use instead of bring in their own tools which can be detected. for instance, if the /usr/bin contains python, the attacker can run scripts, if it contains curl/wget, the attacker can download malware to the system
- /bin is like an emergency kit that the system falls back to in case of need of boot or repair

### Questions for later
- what are code 1 and 2 doing exactly?
```
modprobe loads or unloads kernel modules — think of kernel modules as plugins for the Linux kernel that add hardware support or features.

- sudo modprobe vboxdrv — loads the VirtualBox kernel module. This is what makes VirtualBox actually work — without this module loaded, VirtualBox can't create or run virtual machines. You ran this earlier when fixing your VirtualBox error.
- sudo modprobe -r kvm_amd — the `-r` flag means remove. kvm_amd is the AMD processor virtualization module. Removing it frees up virtualization resources — sometimes needed when VirtualBox and KVM conflict with each other.
```
